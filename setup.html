Cómo conectarlo a Supabase

Esta versión está preparada para guardar registros reales de waitlist con JavaScript usando Supabase. Solo tienes que crear un proyecto, una tabla y copiar dos claves públicas en el archivo de configuración.

Paso 1

Crea proyecto en Supabase

Abre Supabase, crea un proyecto nuevo y copia tu Project URL y tu anon public key. El cliente de JavaScript de Supabase usa esas dos credenciales para conectar desde el frontend. [web:78]

Paso 2

Crea la tabla

En el SQL Editor ejecuta el script de abajo para crear una tabla de registros. Supabase permite crear tablas desde SQL o desde el panel. [web:72]

create table public.waitlist_signups ( id bigint generated always as identity primary key, created_at timestamptz default now() not null, email text not null unique, name text, source text, notes text ); alter table public.waitlist_signups enable row level security; create policy "public_can_insert_waitlist" on public.waitlist_signups for insert to anon with check (true);
Paso 3

Configura el archivo

Rellena supabase-config.js con tu URL, tu anon key y el nombre de la tabla. Después súbelo junto al resto de archivos. Supabase inserta datos en JavaScript con .from(...).insert(...). [web:65]

window.WAITLIST_CONFIG = { supabaseUrl: 'https://TU-PROYECTO.supabase.co', supabaseAnonKey: 'TU_ANON_PUBLIC_KEY', tableName: 'waitlist_signups' };
Paso 4

Publica

Sube también supabase-config.js al hosting junto con index.html, styles.css y app.js. Si quieres un formulario sin backend propio, este enfoque encaja muy bien en una web estática. [web:65][web:78]
